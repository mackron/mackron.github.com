The home page of David Reid - a nobody programmer.

My projects

All of my open source projects are on github and are all public domain.

  • dr_libs: a collection of public domain single-file libraries with minimal dependencies.
    • dr_flac: a library for decoding FLAC files.
    • dr_wav: a library for reading audio data from .wav files.
    • dr_fs.h: a library for loading files from both the native file system and archives such as Zip files, with pluggable back-ends.
    • dr_fsw.h: a library for watching for changes to the file system. Windows only at the moment.
    • dr_path.h: a path manipulation library which never uses the heap.
    • dr_gui.h: a low-level GUI library.
    • dr_vulkan.h: a simple helper library for working with Vulkan.
    • dr_mtl.h: a library for loading material files, compiling them into an intermediate bytecode representation, and then outputting them to shader code such as GLSL.
    • dr_util.h: a utility library for things that don't really fit in any particular category.
    • dr_2d.h: (WIP) a library for drawing 2D graphics.
    • dr_audio.h: (WIP) a simple library for audio playback and recording.

  • dr_ge: (very WIP) a simple "game engine" written in C, mostly from scratch, with a focus on simplicity and ultra light-weightness. Things that make it different:
    • Super simple build system. The entire engine is compiled as a single source file which can be added to your project just like any other.
    • There are no external dependencies except for a few ubiquitous system libraries like pthreads and whatnot.
    • Ultra light-weight with a goal to have executables for simple games (statically linked and including editing tools) to be less than 1MB.
    • A simple, clean, no bullshit API.
    • Public domain.
    • Entirely written in C (no C++).

  • dr_appkit: a toolkit for developing desktop applications on Windows and Linux. Used by dr_ge for it's editing tools.

  • GTGameEngine: a mostly discontinued attempt at a game engine which suffers from not-doing-anything-particularly-different-or-better-itus. See dr_ge which is a similar project with some features that make it quite a bit different from other alternatives. I'm keeping this project around just until I migrate all of the important stuff over to dr_ge.

  • Paddler: it's just Pong. Made in GTGameEngine.

My utilities

These are just simple utility programs I've built which I've found useful for doing very specific things.

  • dr_file2chex.exe: A simple console application which takes a file and converts it's raw data to an unsigned char[] array. I use this to inline small resource files so I can simplify the distribution of whatever it is I'm working on. Output is printed to stdout. Use -n MyVariableName to control the name of the array.
